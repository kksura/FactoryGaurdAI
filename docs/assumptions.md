# Assumptions

Each assumption is numbered so later documents can reference it. If an assumption is invalidated, record the change in `docs/decision-log.md`.

## Environment

- **A1.** The GB10 workstation is the only execution environment available; no Azure subscription, credentials, or Azure CLI are available in this session. Azure work is therefore code + documentation, validated by linting only, and clearly marked as unexecuted.
- **A2.** The developer running this project has sudo-less, non-destructive expectations: no system packages are installed or removed; everything lives in the project venv and Docker containers.
- **A3.** Internet access is available for pulling packages and container images.

## Data

- **A4.** No real production, supplier, or employee data exists or will be used. All data is synthetic, generated by seeded, deterministic code in this repository.
- **A5.** Operator identifiers are pseudonymous (`OP-xxxx`) and are modeled only as process covariates; the system must never be used to score individual workers (see `docs/responsible-ai/`).
- **A6.** Synthetic defect base rates are configurable; defaults target ~2–6% unit-level failure so class imbalance is realistic but trainable at small scale.
- **A7.** Ground-truth causal mechanisms embedded in the generator are a fair proxy for evaluating root-cause ranking; real-world validity would require plant data and is out of scope.

## Modeling

- **A8.** Inspection images are procedurally generated (drawn primitives + controlled noise/perturbations), not photorealistic. They are sufficient to demonstrate the vision pipeline, Grad-CAM, robustness tests, and image-quality drift, but accuracy numbers do not transfer to real camera data.
- **A9.** A GNN is optional; graph-derived features via NetworkX are the supported baseline path (working principle: reliability before sophistication).
- **A10.** The optional LLM summarizer is disabled by default and the system is fully functional without it.

## Azure

- **A11.** Target cloud landing zone follows hub-and-spoke with private endpoints; the enterprise hub (firewall, ExpressRoute, DNS) is assumed to exist and is out of scope — the IaC provisions the spoke.
- **A12.** Microsoft Entra ID tenant, subscription, and a cost approval process exist on the customer side. Region default `eastus2` is a placeholder parameter, not a decision.
- **A13.** Bicep is acceptable as the primary IaC language (ADR-0013); Terraform equivalence is documented but partial.

## Operations

- **A14.** Delayed quality labels arrive within a configurable window (default 7 simulated days); performance-after-labels monitoring assumes this feedback loop.
- **A15.** Human approvers (quality engineer / ML engineer roles) exist for promotion and high-impact recommendations; in local development these are simulated via signed approval records.
